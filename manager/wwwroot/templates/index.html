Management UI for Mailserver Snap

Welcome to your Mail Server, I will try to make this as easy as possible but there is a lot of moving parts in E-mail. If you are stuck feel free to open an issue over at GitHub and ask for help.

How it works

This snap contains the reverse proxy HAProxy that is used to control traffic, this administrative interface and several Ansible playbooks that provisions the actual mail servers. All the source code can be found over at GitHub.

This snap will then use the LXD snap to create several system containers, these containers will then be used to setup all bits and pieces needed to run your mail server.

Why not bundle everything inside the snap?

I started with that idea but it soon got really complicated, most popular mail software has been around quite some time and does not play nice inside a confined environment as a snap package. I finally realized that it would be considerably simpler and cleaner to just use LXD containers.

What do I need?

A server open to the public, try to avoid home servers because many internet service providers (ISP) intended for home use is blacklisted by the big e-mail providers in an effort to try to prevent spam. It is also likely that port 25 is blocked by your ISP for the same reason. A remote smarthost is a work around for this.

A Linux server that supports, and can use the LXD snap, unless you have a custom setup this should not be a problem. If you have no preferences I recommend to pick the latest Ubuntu LTS.

A domain name (or subdomain) that you can update the records on. This assumes some basic DNS knowledge, but to make things easy this site contains all information you need to set everything up.


What this is not

This is never intended to be a complete groupware solution, but I may later on add integrations with Nextcloud with is also available as a snap. Nextcloud offers plenty of nice things, but it's calender and contacts options fits really nice together with e-mail.

What this is

It's based around Postfix and Dovecot and plenty of bits and pieces like OpenDKIM and HAProxy.

Most magic happens in the Ansible playbooks and the python scripts that powers this site. Together they do all the needed magic.

Feel free to contact me at GitHub if you need help, or have suggestions.
